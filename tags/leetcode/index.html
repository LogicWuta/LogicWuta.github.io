leetcode_two-sum

题目

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。

你可以假设每种输入只会对应一个答案。但是，你不能重复利用这个数组中同样的元素。

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/two-sum

常规思路

循环遍历数组，求出能凑出和的值即target-nums[index]，再在数组中寻找这个值。两次嵌套循环，代码如下：
class Solution {
public int[] twoSum(int[] nums, int target) {
int s = nums.length;
for (int i = 0; i < s - 1; i++){
int tmp = target - nums[i];
for (int j = i + 1; j < s; j++){
if (nums[j] == tmp){
return new int[]{i, j};
}
}
}
return null;
}
}

执行通过，用时19ms，内存消耗36.6MB，速度中游水平56.66%，内存消耗较少92.60%的水平。

使用HashMap

思路就是将数组值作为key，下标作为value存放在HashMap里。需要对value进行检查可能为空，因为不能使用同一个数组元素所以还需要加入坐标检查，具体如下。

import java.util.HashMap;

class Solution {
public int[] twoSum(int[] nums, int target) {
HashMap<Integer,Integer> hash = new HashMap<Integer,Integer>();

    int s = nums.length;
    //将数组的值储存为key,坐标存为value
    for (int i = 0; i < s; i++){
       hash.put(nums[i],i);
    }

    for (int i = 0; i < s; i++){
        int tmp = target - nums[i];
        Integer j = hash.get(tmp);
        //查空
        if (j == null){
            continue;
        }
        //检验是否使用同个坐标的元素
        if (j == i){
            continue;
        }
        return new int[]{i, j};
    }
    return null;
}

}

执行结果如下：
执行用时 :4 ms, 在所有 Java 提交中击败了94.73%的用户
内存消耗 :37.4 MB, 在所有 Java 提交中击败了88.54%的用户

效果提升显著且内存消耗增加不多。
